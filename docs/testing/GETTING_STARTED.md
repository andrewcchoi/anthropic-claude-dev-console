# Getting Started with Zero-Gap Testing

This guide walks you through using the comprehensive test strategy on a real feature.

## Quick Start (5 Minutes)

```bash
# 1. Start working on a feature
git checkout -b feature/my-feature

# 2. Generate test checklist
npm run generate-checklist -- src/components/MyComponent.tsx

# 3. Review what tests you need
cat .test-checklist.md

# 4. Generate draft tests (AI-powered)
export ANTHROPIC_API_KEY=your-key
npm run generate-tests -- src/components/MyComponent.tsx

# 5. Write tests in watch mode
npm run test:watch

# 6. Verify completeness before PR
npm run verify-tests
npm test -- --coverage
npm run test:report
```

```typescript
describe('deleteSession', () => {
  it('should remove session from store', () => {
```

#### Step 5: Create Call-Site Audit

If you modified `deleteSession` signature, create an audit:

```typescript
// __tests__/audits/deleteSession-call-sites.test.ts
describe('deleteSession Call-Site Audit', () => {
  it('should verify all call sites pass sessionId parameter', async () => {
    const callSites = await findCallSites('deleteSession');

    for (const site of callSites) {
      const paramCount = extractParameters(site.code);
      expect(paramCount).toBeGreaterThanOrEqual(1);
    }
  });
});
```

The audit runs automatically on pre-commit!

## Common Workflows

```bash

# (edit source files)

# 3. Verify fix
```

```bash

# 3. Make changes incrementally

```

### Reviewing a PR

```bash
# 1. Check test coverage in PR

# 2. Verify checklist complete

# 3. Run tests locally
npm test -- --coverage

# 4. Check for missing audits
npm run test:audits
```

## Tips & Tricks

### Use AI Test Generation Wisely

AI-generated tests are **draft tests**. Always:
1. Review the generated code
2. Add realistic test data
3. Verify edge cases are covered
4. Run the tests to ensure they work

### Call-Site Audits Are Your Safety Net

When refactoring, audits catch:
- Missing parameters
- Wrong parameter order
- Hardcoded values that should be dynamic
- Forgotten call sites

### Test Healing Saves Time

Instead of manually debugging test failures:
1. Run `npm run test:heal`
2. Review the AI suggestions
3. Apply the fixes
4. Verify tests pass

The healer is especially useful after:
- Large refactorings
- API changes
- Dependency updates

### Pre-commit Hook Can Be Bypassed (If Needed)

```bash
git commit --no-verify -m "WIP: incomplete tests"
```

But don't do this on main/feature branches!

### Historical Metrics Track Improvement

The test report saves metrics history:

```bash
cat .test-metrics-history.json
```

Track trends over time:
- Coverage improving?
- Test count growing?
- All layers staying balanced?

## Troubleshooting

### "glob" Module Not Found

The call-site audit test uses native Node.js `fs` module, not `glob`.
If you see this error, ensure you're using the latest version of the audit template.

### AI Commands Fail

Ensure `ANTHROPIC_API_KEY` is set:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
```

Add to your `.bashrc` or `.zshrc` for persistence.

### Pre-commit Hook Not Running

```bash
chmod +x .git/hooks/pre-commit
```

### Coverage Below Threshold

```bash
# Find uncovered code
npm test -- --coverage

# Open HTML report
open coverage/index.html
```

Focus on critical paths first.
